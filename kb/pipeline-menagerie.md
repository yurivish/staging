# Pipeline-shape menagerie

_Auto-generated by `src/toolkit/datapotamus/menagerie_test.clj`._
_To regenerate: `(toolkit.datapotamus.menagerie-test/regenerate-kb!)` from a REPL._

Twenty pipeline shapes covering every glyph and shape kind in
`toolkit.datapotamus.shape` + `render`. Real combinator pipelines
where they exist; hand-built `{:nodes :edges}` topologies otherwise.
Companion to `kb/pipeline-shapes.md` (vocabulary) and
`kb/pipeline-diagrams.md` (glyphs).

Several rendering limits previously documented in `kb/pipeline-diagrams.md` were
surfaced and fixed while building this menagerie. Current behavior:

  - **Branch-exit `↓` marker** (was: rail rows lacked a visual indicator they fed the next stage). Now: each branch's last row gets a left-column `↓`, distinct from in-rail `⎢↓` (in-branch chain flow). See #5, #6, #8.
  - **Prime interior edges** (was: inline `(prime)` clusters listed nodes without edge annotations). Now: `render-inline-prime` runs `classified-annotations`, so interior and boundary-crossing edges show via `⮥`/`⮧`. See #19.
  - **Inline cluster markers attach to a node** (was: `(cycle)` and `(prime)` floated as standalone rows, ambiguous about boundaries). Now: each marker is suffixed onto its first interior member's line. See #15, #19.
  - **`(direct)` is now unambiguous** (was: a `(direct)` row above `sink` could read as attached to the previous branch). Now: branch-exit `↓` on each branch row makes `(direct)` clearly its own branch. See #6.
